# 如何使用创新项目模板（Template Guide）

> 🎯 适合谁：不确定项目结构、第一次使用模板的人
> 📌 本页面解决什么：如何正确使用创新项目模板  
> 👉 阅读后你能做什么：用模板快速、规范地启动项目
> 
本页面将帮助你理解：  
**什么是创新项目模板？为什么要用？以及如何一步一步正确使用它。**

即使你之前从未使用过 GitHub，也可以按照本页面完成。

---

## 🤔 什么是创新项目模板？

创新项目模板（Template Repository）是一个**已经准备好的项目骨架**，里面通常包含：

- ✅ 标准的项目结构
- ✅ 已写好的 README 模板
- ✅ 推荐的目录和文件
- ✅ 一些基础说明和最佳实践

👉 使用模板，你不需要“从零开始”。

---

## ✅ 什么时候应该使用模板？

在以下情况下，**强烈推荐你使用模板**：

- ✅ 第一次创建创新项目
- ✅ 参加 Hackathon / Garage 活动
- ✅ 想快速验证一个想法（PoC）
- ✅ 不确定项目应该包含哪些文件

---

## 🚀 Step 1：从模板创建新项目（最重要的一步）

1. 打开创新 GitHub Organization 首页
2. 找到模板仓库，例如：
   - `template-innovation-repo`
3. 点击 **Use this template**
4. 选择：
   - ✅ Owner：创新 Organization
   - ✅ Repository name：你的项目名称
   - ✅ Visibility：Public（默认）
5. 点击 **Create repository**

✅ 到这里，你已经成功“复制”了一个项目模板。

---

## 📝 Step 2：理解模板里的 README.md（一定要做）

进入你刚创建的项目 Repo，打开 `README.md`。

你会看到一些已经写好的问题或占位内容，例如：

- Problem
- Idea
- Target Users
- Status

👉 **你现在要做的不是写代码，而是回答这些问题。**

### ✅ 建议的填写顺序

1. **Problem**  
   用 2–3 句话说明你想解决什么问题

2. **Idea**  
   描述你的初步解决思路，不用完美

3. **Status**  
   选择当前阶段：
   - idea
   - prototype
   - active

---

## 📂 Step 3：不要急着改结构（新手提示）

模板里已经包含了一些目录和文件。

👉 **新手阶段请遵循以下原则：**

- ✅ 不删除已有文件
- ✅ 不随意改目录名
- ✅ 先把 README 写清楚

> 💡 模板的结构，是为你后续扩展准备的

---

## 🧪 Step 4：如何在模板基础上开始尝试？

当你完成 README 后，你可以：

- 新建一个 `notes.md` 记录想法
- 新建一个 `demo/` 目录放 Demo
- 新建一个 `docs/` 目录放设计说明

✅ 你不需要一次性做好全部内容。

---

## 🚨 常见新手错误（请尽量避免）

- ❌ 还没写 README 就开始写代码
- ❌ 担心“不够成熟”而不敢提交
- ❌ 把模板当成最终成品

👉 模板是起点，不是终点。

---

## 🔄 下一步可以做什么？

完成模板初始化后，你可以：

- 邀请协作者加入 Repo
- 在 Demo Day / Hackathon 中展示
- 根据反馈调整项目方向

如果项目逐渐成熟，可以进入 **毕业（Graduation）流程**。

👉 参见：[项目生命周期](LIFECYCLE.html)
