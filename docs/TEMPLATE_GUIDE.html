📋 如何使用创新项目模板

不需要从零开始——用模板一键启动你的创新项目，跟着 4 步完成初始化。

🎯 适合：第一次用模板的人 ⏱️ 阅读时间：5 分钟
🤔 什么是创新项目模板？

模板（Template Repository）是一个已经准备好的项目骨架，帮助你跳过繁琐的初始化工作，直接进入核心内容。

📁
标准结构
推荐的目录和文件
📝
README 模板
引导你思考问题
📌
最佳实践
内置说明和示例
🚀
即开即用
一键创建，不需配置
✅ 什么时候应该使用模板？
  • 第一次创建创新项目
  • 参加 Hackathon / Garage 活动
  • 想快速验证一个想法（PoC）
  • 不确定项目应该包含哪些文件
💡 简单说：只要你不确定"项目该怎么组织"，就用模板。它是起点，不是限制。
🚀 4 步完成模板初始化
1

从模板创建新项目

打开创新 Organization → 找到 template-innovation-repo → 点击 Use this template → 填写项目名称 → Create repository

✅ 到这里，你已经拥有了一个完整的项目 Repo！
2

填写 README（最重要）

打开 README.md，你会看到几个占位问题。现在要做的不是写代码，而是回答这些问题：

  • Problem — 你想解决什么问题？（2-3 句话）
  • Idea — 你的初步思路是什么？（不需要完美）
  • Status — 当前阶段：idea / prototype / active
3

保持模板结构（新手建议）

模板里的目录和文件是为后续扩展准备的。新手阶段请：

  • ✅ 不删除已有文件
  • ✅ 不随意改目录名
  • ✅ 先把 README 写清楚再动其他
4

开始自由扩展

README 完成后，你可以按需添加内容：

  • 📝 notes.md — 记录想法和灵感
  • 🧪 demo/ — 放置 Demo 代码或截图
  • 📖 docs/ — 设计说明和架构文档
⚡ 做与不做

✅ 推荐

  • 先写 README 再写代码
  • 尽早 Commit，不等完美
  • 邀请同事协作和反馈
  • 记录失败和学到的东西

❌ 避免

  • 跳过 README 直接写代码
  • 担心"不够成熟"不敢提交
  • 把模板当成最终成品
  • 一个人闷头做不交流
🎉 完成检查

如果你已经做到以下几点，说明你正确使用了模板：

  • 使用模板创建了项目 Repo
  • 填写了 README 中的 Problem 和 Idea
  • 标记了当前项目阶段（Status）
  • 做了第一次 Commit
🎯 完成以上步骤后，项目成熟可进入毕业流程，详见 项目生命周期。
